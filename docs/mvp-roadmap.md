# MVP 路线图

## 当前阶段

先写文档。
当前目标是在正式实现前，先对产品形态达成一致。

## 产品阶段视角

为了避免把长期方向误当成首发范围，产品阶段单独定义如下。

### P0：可行性验证版

- 证明 Android 端能够作为 Bluetooth HID 组合设备稳定工作
- 证明手机界面可以承担基础键盘和触控板输入

### P1：首发产品版

- 提供可靠的键盘、触控、滚动、媒体控制体验
- 提供少量预设快捷动作
- 不引入完整 QMK 式可编程特性

### P2：轻量可配置版

- 允许用户调整部分快捷按钮和手势映射
- 让产品从“固定遥控器”进化为“可定制控制面板”

### P3：进阶可编程版

- 在基础体验成熟后，再引入层、宏、tap-hold、one-shot、combo/chord 等高级输入行为

## 工程阶段

## 阶段 0

项目定义与范围确认：

- 明确产品定位
- 定义首版约束
- 勾勒高层架构
- 找到最小但可信的演示版本

## 阶段 1

Bluetooth HID 核心验证：

- Android 应用能够注册为 Bluetooth HID 组合设备
- 桌面端无需额外软件即可配对
- 键盘 report 可发送普通按键与修饰键组合
- 触控区域可驱动相对指针移动
- 基础单击、右击、拖拽、纵向滚动和横向滚动可用

成功标准：

- 至少在一台现代 Windows 设备和一台现代 Mac 上可用
- 重连流程清晰可理解
- 延迟足以支持导航与快捷操作

## 阶段 2

首发产品形态：

- 一个可直接使用的控制界面
- 基础键盘输入
- 触控区域支持单击、右击、拖拽、纵向滚动和横向滚动
- 少量预设快捷动作，例如应用切换、媒体控制、窗口管理、浏览器导航
- 基础连接状态反馈与重连引导

成功标准：

- 用户无需配置即可完成几个真实工作流
- 首发版已经具备“能日常拿来用”的基本完整度

## 阶段 3

轻量可配置层：

- 快捷按钮布局调整
- 预设动作集切换
- 常用快捷键自定义
- 基础手势映射
- 本地持久化配置存储

成功标准：

- 用户无需改代码即可配置一套小型快捷工作流
- 产品已经从固定遥控器升级为可定制控制面板

## 阶段 4

进阶可编程层：

- 基础层，加至少两个附加层
- 宏动作
- tap-hold 行为
- one-shot 修饰键
- combo 或 chord 动作
- 更复杂的手势到动作映射

成功标准：

- 可编程能力已经服务真实工作流，而不只是概念展示
- 高级行为不会破坏基础输入稳定性

## 阶段 5

打磨与产品化：

- 更好的设备配对体验
- 配置预设
- 按应用定制动作集
- 布局导入与导出
- 更成熟的视觉设计

## 在证明必要之前暂缓的事项

- 原生精密触摸板仿真
- 以 BLE HOGP 为中心的架构
- 多设备同步
- 云端备份
- 插件系统

## 第一个值得骄傲的 Demo

一个分成两个区域的手机界面：

- 一个可直接使用的快捷控制区
- 一个支持点击、滚动和拖拽的触控区

这个 Demo 应该已经能支持一些真实工作流，例如：

- 应用切换
- 媒体控制
- 窗口管理
- 浏览器导航
